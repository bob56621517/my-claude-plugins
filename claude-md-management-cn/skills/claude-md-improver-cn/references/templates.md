# CLAUDE.md 模板

## 关键原则

- **简洁**：紧凑、可读的内容；尽可能每个概念一行
- **可操作**：命令应可直接复制粘贴
- **项目特定**：记录该项目独有的模式，而非通用建议
- **最新**：所有信息应反映实际代码库状态

---

## 推荐章节

仅使用与该项目的相关章节。并非所有章节都是必需的。

### 命令

记录使用该项目的基本命令。

```markdown
## 命令

| 命令 | 描述 |
|---------|-------------|
| `<安装命令>` | 安装依赖 |
| `<开发命令>` | 启动开发服务器 |
| `<构建命令>` | 生产构建 |
| `<测试命令>` | 运行测试 |
| `<lint命令>` | lint/格式化代码 |
```

### 架构

描述项目结构，使 Claude 了解各部分的位置。

```markdown
## 架构

```
<根目录>/
  <目录>/    # <用途>
  <目录>/    # <用途>
  <目录>/    # <用途>
```
```

### 关键文件

列出 Claude 应该知道的重要文件。

```markdown
## 关键文件

- `<路径>` — <用途>
- `<路径>` — <用途>
```

### 代码风格

记录项目特定的编码约定。

```markdown
## 代码风格

- <约定>
- <约定>
- <某方案优于某方案>
```

### 环境

记录必需的环境变量和设置。

```markdown
## 环境

- `<变量名>` — <用途>
- `<变量名>` — <用途>

- <设置步骤>
```

### 测试

记录测试方法和命令。

```markdown
## 测试

- `<测试命令>` — <测试内容>
- <测试约定或模式>
```

### 陷阱

记录非显而易见的模式、坑和警告。

```markdown
## 陷阱

- <导致问题的非显而易见事项>
- <顺序依赖或前置条件>
- <应避免的常见错误>
```

### 工作流

记录开发工作流模式。

```markdown

```

## 更新原则

更新任何 CLAUDE.md 时：

1. **要具体**：使用该项目中的实际文件路径和真实命令
2. **要最新**：对照实际代码库验证信息
3. **要简洁**：尽可能每个概念一行
4. **要有用**：这能帮助新的 Claude 会话理解项目吗？
